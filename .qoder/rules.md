# Todoアプリ開発ルール

## プロジェクト固有ルール

### 1. バージョン管理
- Jujutsu (jj) を使用
- コミットメッセージは英語または日本語
- 形式: `type: description`
  - feat: 新機能
  - fix: バグ修正
  - docs: ドキュメント
  - refactor: リファクタリング
  - test: テスト追加・修正
  - chore: その他

### 2. コード品質
- リントエラーは即座に修正
- 未使用の import は削除
- 適切なエラーハンドリングを実装
- ログ出力を適切に管理

### 3. API 設計
- RESTful API の原則に従う
- OpenAPI ドキュメントを常に更新
- レスポンス形式を統一
- ページネーションを実装（一覧取得時）
- any型を極力使用しない


### 4. データベース
- マイグレーションは必ずレビュー
- インデックスを適切に設定
- N+1 問題を避ける
- トランザクション管理を適切に行う

### 5. フロントエンド
- レスポンシブデザインを必須
- アクセシビリティ（a11y）を考慮
- ローディング状態を表示
- エラーメッセージをユーザーフレンドリーに
- Any型を極力使用しない

### 6. セキュリティ
- 機密情報は環境変数で管理
- .env ファイルは .gitignore に追加
- JWT の有効期限を適切に設定
- SQL インジェクション対策（ORM 使用）
- XSS 対策（React の自動エスケープ）

### 7. パフォーマンス
- 不要な再レンダリングを避ける
- API レスポンスをキャッシュ
- データベースクエリを最適化
- 画像やアセットを最適化

### 8. ドキュメント
- コードは自己文書化を心がける
- 複雑なロジックにはコメントを追加
- README は常に最新に保つ
- API 変更時はドキュメントを更新

### 9. テスト
- 新機能にはテストを追加
- 回帰テストを維持
- モックを適切に使用
- テストは独立して実行可能に

### 10. デプロイメント
- Docker イメージを最適化
- 環境変数を適切に管理
- ヘルスチェックを実装
- ロギングを設定

## 禁止事項
- 直書きの機密情報
- 未使用のコードの放置
- エラーの黙り込み
- ハードコードされた設定値
- 適切なテストなしでの本番デプロイ
